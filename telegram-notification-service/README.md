# Telegram Notification Service

Receives DH-HMAC JWT-authenticated ceremony notifications from coordinator and participant scripts and forwards them to a Telegram group chat.

## How it works

Each participant has an X25519 communication keypair from their FROST config. The server has its own X25519 keypair (generated by `npm run frost-notification-config`). When a script sends a notification:

1. The script fetches the server's public key from `/pubkey`
2. The script derives a shared secret via X25519 Diffie-Hellman: `DH(participantPriv, serverPub)`
3. The script creates a JWT HMAC-SHA256'd with that shared secret
4. The server receives the JWT, reads the participant's public key from the payload
5. The server derives the same shared secret: `DH(serverPriv, participantPub)`
6. The server verifies the HMAC — if valid, the participant is authenticated
7. The server forwards the message to Telegram

See [src/dhHmacJwt.ts](src/dhHmacJwt.ts) for the JWT implementation.

## Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/pubkey` | No | Returns the server's X25519 public key |
| POST | `/` | DH-HMAC JWT | Sends a notification to Telegram |
| GET | `/health` | No | Health check |

## Configuration

Environment variables (see [docker/.env.notification-server](../docker/.env.notification-server)):

| Variable | Description |
|----------|-------------|
| `NOTIFICATION_NAMESPACE` | JWT namespace (e.g. `nori-multisig`) |
| `NOTIFICATION_ALLOWED_KEYS` | Comma-separated X25519 public keys from FROST config |
| `NOTIFICATION_SERVER_PRIVKEY` | Server's X25519 private key (generated by `frost-notification-config`) |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token from [@BotFather](https://t.me/botfather) |
| `TELEGRAM_CHAT_ID` | Telegram group chat ID |
| `PORT` | Listen port (default: 3000) |

## Build

```sh
./build.sh
```

Builds multi-arch images (amd64 + arm64) using `docker buildx`.

## Release

```sh
./release.sh
```

Pushes arch-specific images and creates multi-arch manifests for both the versioned tag and `latest`.
